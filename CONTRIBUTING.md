# Contributing

Thanks for your interest in Shortly!

## Issues

- Search existing issues before opening a new one.
- Provide steps to reproduce, expected behavior, and actual behavior.
- Include relevant environment details (Go version, OS, database driver).

## Pull requests

1. Fork the repo and create a branch from `main`.
2. Run the tests and make sure they pass.
3. Keep changes focused — one feature or fix per PR.
4. Write a clear commit message explaining the *why*, not just the *what*.
5. Update docs if your change affects the API, config, or architecture.

## Development setup

See the [README](README.md#getting-started) for setup instructions.

## Code style

- **Go**: follow `gofmt` and standard Go idioms.
- **JavaScript/React**: follow the existing patterns in the codebase.
- No commented-out code in PRs.
